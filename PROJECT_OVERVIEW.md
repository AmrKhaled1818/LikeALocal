# LikeALocal — Project Overview

A social Flutter app where users discover and share **hidden gem** locations, explore them on a map, chat with friends, and get AI-powered recommendations and trip plans.

---

## 1. Tech Stack

| Category | Tool | Why / How Used |
|---|---|---|
| **Framework** | Flutter (Dart 3.5+) | Cross-platform mobile — primary target is Android |
| **Auth** | Firebase Auth | Email/password + Google Sign-In; `authStateChanges` stream drives routing |
| **Database** | Cloud Firestore | Real-time NoSQL; offline persistence enabled with unlimited cache |
| **Push Notifications** | Firebase Messaging (FCM) | Push for upvotes, comments, DMs; token stored per user |
| **Media CDN** | Cloudinary | Image + video upload and delivery; chosen for CDN speed and auto-transformations |
| **AI (LLM)** | Groq API | Chat assistant, trip planning, post summaries, tags, best-time hints |
| **AI (Vision)** | OpenRouter + Nemotron VL | Image description generation in CreatePostScreen |
| **Maps** | flutter_map + Stadia Maps | Interactive map with clustered markers; Stadia provides tile imagery |
| **State** | Provider (ChangeNotifier) | 6 global providers injected at app root via MultiProvider |
| **Navigation** | go_router | Declarative routing; `StatefulShellRoute` for 5-tab bottom nav |
| **Background** | WorkManager | Proximity check task every 15 min even when app is closed |
| **Location** | geolocator + geocoding | GPS position for AI chat, trip planner, proximity alerts; reverse geocoding for address text |
| **Local Storage** | shared_preferences | Persists mood, AI usage count, theme, proximity cache |
| **HTTP** | http package | REST calls to Groq and Cloudinary |
| **Media Picker** | image_picker | Pick images from gallery/camera or pick a video |
| **Video** | video_player | Inline video playback on post detail and chat |

| **Connectivity** | connectivity_plus | Detects offline state; triggers banner |

## 2. Architecture

```
Screen / Widget
    ↓  context.watch / Consumer / Selector
Provider  (holds state, calls repos, notifies listeners)
    ↓
Repository  (raw Firestore/Auth calls only — no business logic)
    ↓
Firebase / Cloudinary / Groq API
```

**Rules:**
- Screens never call repositories directly — always through a Provider
- All mutable state lives in Providers
- **Optimistic updates**: state changes locally first, Firestore write second; reverts on error (used for upvotes and saves)

## 3. App Startup

`main()` runs in order:
1. `Firebase.initializeApp()` — reads `firebase_options.dart` (auto-generated by FlutterFire CLI)
2. Firestore configured: `persistenceEnabled: true`, `cacheSizeBytes: UNLIMITED` → offline works
3. `NotificationService.initialize()` — FCM listener + local notification channels
4. `ProximityService.setup()` — registers WorkManager background task
5. `runApp()` → `MultiProvider` with 6 providers → `GoRouter`

The router is built once in `initState`, never in `build`, to avoid a GlobalKey crash on rebuilds.

## 5. Data Models

### PostModel
The core content unit. Key fields:

- `postId`, `userId`, `username`, `userAvatarUrl`, `isSuperUser`
- `title`, `description`, `localTips`, `recommendedDishes[]`
- `imageUrls[]` (up to 5, canonical) + `imageUrl` (legacy, kept for backward compat)
- `videoUrl` / `videoPublicId` — mutually exclusive with images
- `lat`, `lng` — GPS (0.0 = not pinned); `location` — text
- `category` — Restaurant / Café / Mall / Park / Cultural / Viewpoint / Shop
- `upvotes`, `downvotes`, `upvotedBy[]` — prevents double-voting
- `checkinsByHour{}` — hour → count; `lastCheckinAt`; `bestTime` (AI-cached phrase)
- `aiSummary` — AI-generated 2-sentence summary

### UserModel
- `uid`, `username`, `avatarUrl`, `bio`, `karma`, `isSuperUser`
- `preferences: {budget, atmosphere, favCategories[]}` — drives Vibe Match
- `fcmToken` — updated on every login

### MessageModel
- `senderId` = `'ai'` for AI messages; `type` = `'ai_response'`
- `readBy[]` — UIDs for read receipts
- `videoUrl` — optional video message

### CommentModel
- `parentId` = null (top-level) or set (reply)
- Author info denormalised at write time

### PlaceGroup *(client-computed, not in Firestore)*
Groups posts about the same real-world place. Grouping criteria (any one match):
1. Normalised titles match (strips "restaurant", "café", etc.)
2. Exact location strings match
3. Both posts have GPS within 150 m

`PlaceGroup.groupPosts(posts)` → sorted by `totalUpvotes` desc. Used in Search and PlaceDetailScreen.

### TripStop
Output of trip planner: `postId`, `stayMinutes`, `note` (travel note from previous stop).

## 6. Firestore Structure

    comments/{commentId}

    savedPosts/{postId}

AI chat ID = `ai_{userId}`; DM chat ID = deterministic from sorted UIDs.

## 8. Providers

| Provider | Responsibility |
|---|---|

| **ConnectivityProvider** | Streams connectivity; triggers offline banner |

## 9. Features & How They Work

### Feed & Vibe Match
PostsProvider loads 10 posts per page with a Firestore cursor. When the user scrolls within 300 px of the bottom, `loadMore()` appends the next page. A separate real-time stream (`getFeedPosts()`) keeps the full post list live for the map.

`rankedFeed(prefs)` returns a **sorted copy** — never mutates the paginated list so the cursor stays valid:
1. Super User posts first
2. Vibe Match score (`VibeScore.forPost`) — see section 10
3. Recency

Shimmer skeleton shows while loading. `AutomaticKeepAliveClientMixin` preserves scroll position when switching tabs.

---

### Creating a Post
1. `_checkDailyLimit()` reads SharedPreferences (`post_count_{uid}`, `post_reset_{uid}`) — free users get 3 posts/day; hits → PaywallSheet
2. User fills form: title, category, description, tips, dishes (Restaurant/Café only)
3. User picks up to 5 images **or** 1 video (mutually exclusive via `image_picker`)
4. "Generate AI description" → VisionService sends image to OpenRouter Nemotron VL → pre-fills description
5. Optional GPS pin via `LocationPickerScreen` (flutter_map tap)
6. Submit: images compressed to 1080×1080 q72 → `CloudinaryService.uploadImages()` (concurrent via `Future.wait`) → `PostsRepo.createPost()` → +10 karma → `AIService.generatePostSummary()` runs async and caches result

---

### AI Chat Assistant
`ConversationScreen` detects `chatId.startsWith('ai_')`. On send:
1. Checks daily limit (SharedPreferences `ai_count_{uid}` — reset every 24 h; Super Users bypass)
2. Gets GPS via geolocator; builds post list from `PostsProvider.allPosts`
3. Calls `AIService.getAIResponse(history, user, lat, lng, availablePlaces)` — Groq API with system prompt containing user preferences + full post list
4. AI is instructed: only recommend from app's post list, filter by requested area/type
5. Response rendered via `flutter_markdown`; `_findRelatedPosts()` scores posts against the reply — any scoring ≥ 6 pts shown as tappable chips

---

### Trip Planner
Form inputs: time (1–4 h), mood, budget, transport, group size, time of day, optional GPS start.

On "Build My Trip":
1. All posts with valid GPS collected as candidates (up to 30 serialised as JSON)
2. `AIService.planTrip()` → Groq returns `[{id, stayMinutes, note}]` JSON
3. If empty → `greedyItinerary()` fallback (nearest-neighbour, haversine distance, mood-weighted, time-budgeted)
4. Results: `FlutterMap` with `PolylineLayer` + numbered markers; `CameraFit.coordinates` auto-fits bounds; scrollable stop timeline with VibeBadge + CrowdBadge per stop

---

### DM Chat — Typing & Read Receipts
- **Typing**: first keystroke → `ChatRepo.setTyping(true)` writes a Timestamp to Firestore; 3 s timer resets it to false; `watchTyping` streams `true` if other user's timestamp is < 6 s old → animated `_TypingBubble` shown
- **Read receipts**: `markRead` adds uid to `readBy[]` of all messages; own messages show grey ✓ (sent) → blue ✓✓ (read)
- AI chat reuses the same typing bubble ("AI is thinking…") driven by a local `_sending` flag — no Firestore writes

---

### Crowd Indicator (CrowdBadge)
`crowdInfo(post)` picks the best available signal in priority order:
1. `lastCheckinAt` < 2 h → **"Busy right now"** (pulsing dot)
2. `checkinsByHour` total ≥ 3 → peak hour → **"Liveliest around X PM"**
3. `bestTime` not empty → cached AI phrase
4. Category heuristic → generic phrase

`CrowdBadge(autoGenerate: true)` lazily calls `AIService.generateBestTimeHint()` (Groq fallback model, ≤ 6 words) if no cached value, then writes it via `updateBestTime()` — never regenerated.

---

### Proximity Alerts (Background)
WorkManager runs `callbackDispatcher` every 15 min (even when app is closed). The isolate cannot access Firebase, so:
- When user saves a post → `PostsProvider.cacheSavedPosts()` writes `[{lat, lng, title}]` JSON to SharedPreferences
- Isolate reads from SharedPreferences → gets GPS via geolocator → computes haversine distance per saved post → if < 500 m and 30 min since last alert → fires local notification

---

### PlaceGroup & Place Detail
`PlaceGroup.groupPosts(posts)` runs client-side: groups posts about the same real place by title normalisation, location string, or GPS proximity (< 150 m). Used in SearchScreen's Places tab.

`PlaceDetailScreen` receives a `PlaceGroup` via `Navigator.push` and shows:
- `SliverAppBar` with cover image
- `PageView` across all posts in the group
- `_MergedReviews`: subscribes to comment streams for **all** posts in the group, merges and sorts newest first; each review shows which post it belongs to

## 10. Vibe Match Scoring

`VibeScore.forPost(post, prefs, {mood})` → int 15–100

```
Base:                          +30
Favourite category match:      +22
Atmosphere keyword hits:       +4 each, max +12
Budget keyword match:          +6
Active mood — category match:  +28  (dominant signal)
Active mood — wrong category:  -12
Active mood — keyword hits:    +5 each, max +18
Content completeness:          up to +12
Popularity nudge:              post.upvotes clamped to 0–10
Floor / Cap:                   min 15, max 100
```

Mood acts as a soft category filter because the ±28/12 swing is large enough to visibly reorder the feed.

**Labels:** ≥ 85 = "Top match" · ≥ 70 = "Great fit" · ≥ 55 = "Good fit" · < 55 = "Worth a look"

## 11. Karma & Super User

**Karma sources:** create post +10 · post upvoted +2 · comment +1 · DM sent +3 · check-in +1

**At karma ≥ 100** (`UserRepo.addKarma`):
- Sets `isSuperUser: true` on user doc + all their posts (batch write)
- Writes a `superuser` notification → `AuthProvider` fires a local push

**Super User benefits:** amber badge · posts ranked first in all feeds · AI daily limit bypassed

## 12. AI Service — Groq API

All calls go through `AIService._complete()`:
- Up to 3 attempts; on 429 from primary → retries once on fallback model; on 5xx → exponential backoff
- All calls log token counts and rate-limit headers to `lastStats`

| Method | Model | Output |
|---|---|---|
| `getAIResponse` | Primary | Conversational string (rendered as markdown) |
| `generatePostSummary` | Primary | 2-sentence post summary |
| `generateBestTimeHint` | Fallback | ≤6-word phrase |
| `planTrip` | Primary | `List<TripStop>` from JSON |
| `parseVibeQuery` | Fallback + JSON mode | `{category, atmosphere, keywords}` |
| `generateTags` | Fallback + JSON mode | `List<String>` of 4–8 tags |
| `generateLocalTip` | Primary | One-line insider tip |

**AI daily limit:** tracked in SharedPreferences (`ai_count_{uid}`, `ai_reset_{uid}`). Resets every 24 h. Clearing chat does **not** reset it. Super Users bypass.

---

## 13. Media Upload

Images: picked via `image_picker` → compressed to 1080×1080 q72 → `CloudinaryService.uploadImages()` runs all uploads concurrently via `Future.wait` → returns `{imageUrl, imagePublicId}` pairs → stored in `PostModel.imageUrls[]`.

Videos: `ImagePicker.pickVideo(gallery)` (mutually exclusive with images) → `CloudinaryService.uploadVideo()` with `resourceType: Video` → stored in `PostModel.videoUrl`.

Deletion: `PostsRepo.deletePost()` calls Cloudinary to delete by `imagePublicId` before removing the Firestore doc.

## 14. Notifications

| Type | Package | Trigger |
|---|---|---|
| Instant toast | toastification | Post created, saved, upvoted |
| Local async | flutter_local_notifications | Comment, DM, Super User promotion, proximity alert |
| Remote push | Firebase Messaging (FCM) | Same events from server |

FCM token is saved to `users/{uid}.fcmToken` on every login. Notification documents live at `notifications/{uid}/items/{notifId}` with fields: `type`, `body`, `createdAt`, `read`.

## 15. Offline Support

- Firestore `persistenceEnabled: true` + unlimited cache → all loaded data available offline
- Writes queue locally and sync when connection restores
- `ConnectivityProvider` shows a red "No internet" banner via `TopBar` when offline
- Seed places (`seed_places.dart`) are hardcoded curated posts merged into the feed — always visible on the map regardless of connectivity or pagination state

---

## 16. Security

- All API keys in `lib/core/constants/app_config.dart` — listed in `.gitignore`, never committed
- All Firebase calls wrapped in `try/catch`
- Firebase Auth errors mapped to readable strings (never shown raw)
- "Get Directions" uses text query `"title, location"` — not raw GPS — so Google Maps finds the actual entrance
